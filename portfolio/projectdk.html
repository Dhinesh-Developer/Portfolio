Last Try Food App

Feature Description
Project Overview The Last Try Food App is a full-stack web application designed to replicate the core functionalities of popular food delivery platforms like Swiggy. It provides users with a seamless experience for browsing restaurants, exploring menus, placing orders, and tracking food deliveries. The application is built using modern web technologies including HTML, CSS, JavaScript, Bootstrap, J2EE, JSP, JDBC, and MySQL.
Key Features
  • User-Friendly Interface: Built with HTML, CSS, JavaScript, and Bootstrap for a responsive design.
  • Backend Functionality: Java Servlets and JSP for real-time dynamic content rendering.
  • Database Management: Uses MySQL for storing user data, restaurant information, menu details, and order history.
  • Admin Dashboard: Allows admins to manage restaurant data, menu items, and user information.
  • Interactive UI: Smooth animations and cross-device compatibility.
  • Authentication System: Secure user registration, login, and error handling.
  • Add to Cart & Payment: Users can add items to the cart and proceed to payment.
  • Order History Tracking: Users can view and track past orders.
Tech Stack
  • Frontend: HTML, CSS, JavaScript, Bootstrap.
  • Backend: Java (J2EE), Servlets, JSP, JDBC.
  • Database: MySQL for relational data storage.
How It Works
  • Homepage: Displays featured restaurants, reviews, and quotes with easy navigation.
  • Restaurant Page: Displays restaurant information with a card layout.
  • Menu Page: Lists menu items with prices and quantities to add to the cart.
  • Cart and Payment: Users view their cart, total price, and proceed to checkout.
  • Order History: Displays past order details including restaurant name and total spent.
Database Structure
  • Users Table: userId, name, username, password, email, phone, address, role, createdDate.
  • Restaurants Table: restaurantId, restaurantName, address, rating, eta, imagePath.
  • Menu Table: menuId, restaurantId (foreign key), itemName, price, ratings, isAvailable.
  • Order Table: orderId, userId (foreign key), restaurantId, totalAmount, orderDate.
